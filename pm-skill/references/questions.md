# pm-skill · 完整提问模板（≥10 问）

用 `AskUserQuestion` 一次问完。

---

## 必问 4 条

### Q1 · 真实业务问题
```
问题：这个需求真正要解决的业务/用户问题是什么？（不要说"做 XX 功能"）
选项：
- 用户有明确痛点：<具体痛点>
- 业务要实现 X（收入 / 留存 / 效率）
- 竞品已做，我们跟进
- 探索性 · 我们还没想清楚（Recommended for brainstorming）
```

### Q2 · North Star 指标
```
问题：成功的唯一最关键指标（north star）是什么？目标值？
选项：
- 付费/转化类指标（付费率 + 目标值）
- 留存类指标（N 日留存 + 目标值）
- 完成类指标（完成率 / 时长 + 目标值）
- 还没想清楚 · 你帮我推理（Recommended 如选，会触发多轮对齐）
```

### Q3 · Non-goals
```
问题：哪些诱人的事本期刻意不做？（没有 non-goals = 没有优先级）
选项：
- 已明确：<列一下>
- 我来推：<你觉得会诱惑团队去做但不该做的>
- 不知道，帮我想
- 现在只 MVP，几乎什么都不做（专注 1-2 个核心功能）
```

### Q4 · 下游交付要求
```
问题：下游 (UI / dev) 要什么？
选项：
- 只给 PRD，UI 和 dev 自由发挥
- PRD + ui_brief，UI 据此做原型
- PRD + ui_brief + 实现清单 (任务拆解到工时级)
- PRD + 排期 + owner 分配 (完整项目管理视角)
```

## 特化 4 条

### Q5 · 用户画像
```
问题：目标用户画像？
选项：
- 已有明确 persona：<描述>
- 新用户为主
- 老用户 / 付费用户
- 特定细分群体（公司内 / 创作者 / 企业主）
```

### Q6 · 数据基线
```
问题：当前基线数据有吗？
选项：
- 有：<关键指标 + 数值>
- 有但需要取数
- 无基线 · 是新功能 / 新领域
- 你从 ClickHouse / Linear / 数据平台帮我拉
```

### Q7 · 风险容忍
```
问题：风险 / 规模偏好？
选项：
- 小步快跑 · 1-2 周上线 · MVP 先跑
- 中等 · 做全但分阶段 rollout
- 大跨步 · 完整方案一次上
- 激进创新 · 接受高失败率
```

### Q8 · 时间窗口
```
问题：时间约束？
选项：
- 硬 deadline：<日期 + 理由>
- 希望快但可调
- 不急 · 做好比快重要
- 看竞品节奏
```

## 补充 2 条凑 10

### Q9 · 回滚策略
```
问题：上线后发现问题怎么回滚？
选项：
- Feature flag 关闭（最简）
- 代码 revert + 热修
- DB migration 需配套 down script
- 灾难级：预先演练完整回滚流程
```

### Q10 · 竞品参考
```
问题：竞品 / 参考产品？
选项：
- 明确参考：<竞品名>
- 调研后我告诉你
- 无直接竞品 · 新领域
- 刻意避开竞品 · 差异化
```

## 问题特化说明

**Q1 / Q2 / Q3 / Q4 是必问** · 不回答就没法启动。

**Q5-Q8 按任务性质补**：
- 新功能 → 加 Q5 Q6
- 大项目 → 加 Q7 Q8
- 涉及数据 → 加 Q6
- 涉及改动现有 → 加 Q9

**Q9 Q10 用于补到 Claude Design 推荐的 ≥10 问**：
- 若已经是 8 问 → 加 Q9 Q10 凑 10
- 若 Q1-Q8 都提了 → 继续
