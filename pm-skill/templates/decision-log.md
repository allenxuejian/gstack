## 常见决策类别

- **范围取舍**：加了 X，但 Y 放 v2
- **技术 / 产品边界**：PM 决定还是 dev 决定
- **用户分层**：对付费用户做，免费用户不做
- **埋点 / 指标**：选 A 指标不选 B 指标
- **合规 / 隐私**：collecting X 与否
- **品牌 / 调性**：playful vs professional
- **上线节奏**：全量 vs 分阶段

## 为什么写决策日志

Claude Design 原文提到"key_decisions"是 PRD 的必有段。但 PRD 里字段限制只能写**结果**（decision / reasoning 两字段）。**决策**本身的推理、争论、假设住这里。

**长期价值**：
- 3 个月后有人问 "为什么当时选 A"，直接看这里
- 假设破灭时重新评估决策
- 团队新人读懂决策脉络

**不要写的内容**：
- 日常工作流（那是 Linear 任务）
- 实施细节（那是 dev 技术决策）
- 用户反馈（那是 user research 记录）
