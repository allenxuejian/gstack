# 决策矩阵模板

## 1. 维度选择

**通用维度**（按需挑 ≥4 个）：
- 性能（吞吐 / 延迟 / 资源）
- 心智成本（学习曲线）
- 生态（star / 下载量 / 贡献者 / 维护者活跃度）
- 维护性（文档质量 / 可调试性 / 演进稳定性）
- 成本（license / infra / 人力）
- 兼容性（与现有栈）
- 社区支持（Stack Overflow / GitHub Issues 响应）
- 安全性（CVE 历史 / 审计）
- 供应商锁定（开源 / 可切换）

---

## 2. 权重分配

加起来 = 100%。由 Q2 约束推导：
- 性能敏感场景 → 性能 > 40%
- 团队小 → 心智成本 + 维护性 > 50%
- 成本严控 → 成本 > 30%

---

## 3. 打分标准

每个维度 1-10：
- 1-3：不及格
- 4-6：可用
- 7-8：良好
- 9-10：优秀

**引用要求**：每个 ≥ 7 分必须 cite 证据。

---

## 6. 敏感性分析

"如果把权重从 A 改成 B，推荐会变吗？"

- 当前推荐：Option A (8.25)
- 亚军：Option B (7.35) · 差距 12%
- 敏感维度：如果"心智成本"权重从 25% 提到 40%，B 会翻盘

---

## 7. Trade-off 明示

> Option A 在**性能**和**成本**上最优，但**心智成本**比 B 高。选 A 等于"接受 2 周学习成本换长期运行优势"。
